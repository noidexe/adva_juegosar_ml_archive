De un thread the la lista de la GDC de alguien recomendando algunas 
aplicaciones OpenSource.


===

There are Open Sources solutions for so many things nowadays. The nice
thing about Open Source is a lot of times you get cutting edge (and even
better than very expensive commercial products) products that don't cost
you anything and if you choose, you can have a say in or even be a part
of their future development. Here are programs that I use regularly
(keep in mind I'm not an artist nor an audio guy) that might be helpful
in what you're doing:

OpenOffice (http://www.openoffice.org): Great fully featured office
package that will also read and write Microsoft office files.

GIMP (http://www.gimp.org): The Gnu Image Manipulation Program is great
as an alternative to Photoshop and it will read and write Photoshop
files. They do have an animation module but I haven't tried it yet.

Audacity (http://audacity.sourceforge.net): This is a great program for
working with audio files. It's served all my recording needs.

Miranda Instant Messenger (http://www.miranda-im.org): Nice instant
messenger that supports AIM, MSN, Yahoo, ICQ, IRC, Jabber, Gadu-Gadu,
with e-mail notifications for those and Gmail. Not sure if that is of
use to your kids though.  :)

Well, lots more Open Source that I use (including a bunch for software
development) and even more that is available. You can check out the site
for Software Freedom Day and see some links there to things like OpenCD
which is a whole CD of opensource applications.
(http://softwarefreedomday.org/index.html)

As for a Java Script tutorial, you can find great tutorials for about
every language there is through a google search. I've learned a few
languages that way myself. This one looks okay:
http://www.pageresource.com/jscript/

Hope that helps.

===

Oracle realmente es un seï¿½or RDBMS, pero por cuestiones presupuestarias 
muy fuera de alcance para la mayoria, asi que lo dejaria afuera de la 
cuestiï¿½n.
Yo si estoy en un proyecto MMORPG (Regnum Online), y debido a cuestiones 
de presupuesto, decidimos utilizar mySQL y PostgreSQL, y la experiencia 
hasta ahora ha sido mas que satisfactoria, ha cubierto todas nuestras 
necesidades.
Si bien, ambas bases de datos no tienen todas las features de los 
motores mas grandes, tienen la cantidad suficiente (y quizas de sobra en 
algunos casos) como para hacer cualquier tipo de juego.
Ms SQL Server, si bien es propietario, no es relativamente caro y ofrece 
tambien bastantes features muy buenas (siendo los stored procedures, uno 
de los mejores).

Las features que yo buscaria en un RDBMS para un MMORPG, serian las 
siguientes:
- Transacciones (seguridad, rollback)
- Stored Procedures/Functions (performance, centralizaciï¿½n)
- Joins: Inner, left, right, outer (flexibilidad, aunque este lo tienen 
casi todos)
- API's robustas, con bindings a varios lenguajes y documentacion
